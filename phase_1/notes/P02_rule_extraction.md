# Paper ID: P02

## 1. Basic information

- Title: GUARDIAN: Safeguarding LLM Multi-Agent Collaborations with Temporal Graph Modeling
- Year: 2025
- Venue / Source: NeurIPS 2025 / arXiv:2505.19234v2
- Main problem:
  Multi-agent LLM systems can amplify hallucinations or propagate injected errors across agents and communication rounds. Existing defenses often check individual outputs or use majority voting, but they do not explicitly model how misinformation/error flows through the collaboration graph.
- Main contribution:
  GUARDIAN models multi-agent collaboration as a discrete-time temporal attributed graph. Nodes represent agents at each timestep, edges represent inter-agent communication, and node attributes encode agent responses. It uses an unsupervised encoder-decoder graph model to detect anomalous nodes/edges, then mitigates risk by removing anomalous agents or corrupted communication paths. It also uses graph abstraction based on Information Bottleneck Theory to compress interaction graphs while preserving essential anomaly-detection patterns.

## 2. Agent trace / event abstraction

Paper uses which type of trace?

- Tool-call event: No. The paper does not focus on external tool calls; the main unit is inter-agent communication and agent response.
- Message-action trace: Yes. The paper models multi-round agent responses and messages passed between agents.
- State transition: Yes. The collaboration evolves over timesteps, represented by temporal graph sequence `G_t = (V_t, E_t, X_t)`.
- Action trajectory: Partial. It is not a tool/action trajectory, but a collaboration trajectory across rounds.
- Span / observability trace: No.
- Other: Temporal attributed graph; anomaly graph; node/edge reconstruction trace.

Important fields mentioned:

- action name: Not explicit as tool action. Relevant event types are `agent_response`, `inter_agent_message`, `communication_edge`, `anomaly_detection`, `node_removal`.
- arguments: Input query `q`, agent prompt `p_i`, predecessor responses `R_{t-1}`, current response `r_{t,i}`.
- output: Agent response `r_{t,i}`, reconstructed node attributes, reconstructed graph structure, final answer/consensus.
- state: Temporal graph state `G_t = (V_t, E_t, X_t)`, node embeddings `Z_t`, collaboration network before/after anomaly removal.
- timestamp / order: Timestep `t`, discussion round, temporal order from `t-1` to `t`.
- source / provenance: Directed edges `(v_{t-1,i}, v_{t,j})` encode which previous agent response is used as context by which later agent.
- approval: No human approval mechanism.
- error / status: Hallucination indicator `h_{t,i}`, error indicator `err_{t,i}`, anomaly score `s_v`, false discovery rate, anomaly detection rate.
- user instruction: Query `q` is the original task/question given to the multi-agent system.

## 3. Verification / policy / rule

Extracted rules:

### Rule 1

- Original description:
  Hallucinated information generated by one agent may propagate to downstream agents in later rounds, causing hallucination amplification.
- Normalized rule:
  WHEN an agent response at timestep `t` is anomalous or hallucinated
  AND the response influences downstream agents through communication edges
  REQUIRE the anomalous node and its outgoing propagation path to be detected and removed before subsequent collaboration rounds
  ELSE hallucination propagation risk.
- Rule class:
  Temporal propagation rule; provenance rule; anomaly-detection rule; recovery/governance rule.
- Trigger:
  New collaboration round or new agent response generated.
- Condition:
  Node attribute reconstruction error is high, or anomaly score `s_v > threshold`, or the response deviates from normal semantic/structural collaboration patterns.
- Required evidence:
  agent id, timestep, agent response text, node attribute embedding, incoming/outgoing communication edges, anomaly score, propagation path, previous-round responses, current graph state.
- Verdict / enforcement:
  Verdict = anomalous node / unsafe propagation path. Enforcement = remove anomalous agent node and incident edges from subsequent graph iterations.
- Possible backend:
  Python graph monitor; temporal graph anomaly detector; provenance/taint monitor; LTL wrapper for the high-level rule `G(anomaly_detected -> removed_before_next_round)`.

### Rule 2

- Original description:
  In agent-targeted error injection, an adversary directly corrupts one agent through malicious prompts or compromised behavior, forcing that agent to generate erroneous output.
- Normalized rule:
  WHEN an agent has high anomaly score at timestep `t`
  AND the anomaly is localized to that agent's response attributes
  REQUIRE the compromised agent node to be excluded before later agents consume its output
  ELSE error propagation risk.
- Rule class:
  State/attribute anomaly rule; temporal rule; recovery/governance rule.
- Trigger:
  Agent response produced at a new timestep.
- Condition:
  Attribute reconstruction error for the node is high, or the node is ranked as highest anomaly by GUARDIAN.
- Required evidence:
  agent id, timestep, response text, response embedding, original node attributes `X_t`, reconstructed attributes `X_hat_t`, reconstruction error, anomaly score, downstream recipients.
- Verdict / enforcement:
  Verdict = compromised/anomalous agent. Enforcement = delete the node from the updated collaboration network and remove its connected edges.
- Possible backend:
  Python monitor over graph state; graph anomaly detector; SMT/Python rule over thresholded anomaly score; temporal monitor.

### Rule 3

- Original description:
  In communication-targeted error injection, an adversary corrupts the message transmitted from one agent to another between rounds, so later agents receive modified information.
- Normalized rule:
  WHEN a communication edge `(v_{t-1,i}, v_{t,j})` carries inconsistent or anomalous information
  AND the structural reconstruction error or edge anomaly score is high
  REQUIRE the corrupted communication edge to be removed or the affected node to be excluded
  ELSE corrupted information may propagate.
- Rule class:
  Communication integrity rule; provenance rule; edge anomaly rule; taint rule.
- Trigger:
  Inter-agent message transmission between timestep `t-1` and timestep `t`.
- Condition:
  Structural reconstruction error for the edge is high, or the receiver response becomes anomalous due to corrupted predecessor context.
- Required evidence:
  sender agent id, receiver agent id, timestep, edge id, original message, received/modified message, communication adjacency, structural reconstruction error, causal/provenance link.
- Verdict / enforcement:
  Verdict = anomalous/corrupted edge or affected node. Enforcement = prune the edge, remove affected node, or exclude the corrupted information from later rounds.
- Possible backend:
  Provenance graph monitor; Python edge-integrity checker; graph anomaly detector; taint monitor.

### Rule 4

- Original description:
  Dense multi-agent communication can contain redundant/noisy information. GUARDIAN uses Information Bottleneck-based graph abstraction to compress the graph while preserving essential temporal-structural patterns for anomaly detection.
- Normalized rule:
  WHEN compressing a temporal interaction graph
  REQUIRE task-relevant temporal, structural, and attribute patterns to be preserved
  AND redundant/noisy interaction information to be reduced
  ELSE anomaly detection may lose necessary evidence.
- Rule class:
  Evidence preservation rule; abstraction quality rule; provenance preservation rule.
- Trigger:
  Graph preprocessing / graph abstraction before anomaly scoring.
- Condition:
  Compression must preserve node response semantics, communication structure, and temporal dependency needed for anomaly detection.
- Required evidence:
  original graph `G_t`, node attributes `X_t`, edge set `E_t`, compressed representation `Z_t`, target/collaboration outcome `Y_t`, information-flow relation, reconstruction loss.
- Verdict / enforcement:
  Verdict = preserved / insufficient abstraction. Enforcement = use compressed graph only if essential anomaly patterns are preserved.
- Possible backend:
  Preservation checker; Python evidence-completeness checker; information-flow monitor; ablation-based verifier.

## 4. What can be reused for our work?

- Abstraction reused:
  Temporal attributed graph abstraction:
  `event/trace = (agent_id, timestep, response, incoming_edges, outgoing_edges, graph_state, anomaly_score)`.
  This is useful for representing multi-agent message-action traces and causal/provenance links.

- Rule reused:
  1. Detect and remove anomalous response before it propagates.
  2. Detect and remove compromised agent node.
  3. Detect and remove corrupted communication edge.
  4. Preserve temporal/provenance evidence during graph abstraction.

- Evidence reused:
  agent id, timestep, response text, response embedding, communication edge, predecessor response, anomaly score, reconstruction error, graph state, removed node/edge, propagation path.

- Limitation:
  GUARDIAN is strong for multi-agent communication safety, hallucination propagation, and error propagation, but it is not a symbolic verifier. Rules are learned through graph reconstruction/anomaly scoring rather than explicit DSL/LTL/SMT contracts. It also does not focus on tool calls, approval policies, typed tool arguments, or external side effects.

## Extraction note for our project

P02 should be used as the anchor paper for **trace-based assurance / replay / temporal graph monitoring** and partly for **provenance / taint / data-flow safety**.

Main insight:

`multi-agent collaboration trace = temporal graph of agent responses + communication edges`

Map to our normalized event stream:

```text
raw multi-agent dialogue
→ event(agent_response, agent_id, timestep, response_text)
→ event(message_edge, sender, receiver, timestep, input_ref, output_ref)
→ event(anomaly_score, node_id/edge_id, score, reason)
→ event(governance_action, remove_node/remove_edge, target, timestep)
```

Map to Rule IR:

```text
WHEN node_or_edge anomaly score exceeds threshold
AND propagation path exists
REQUIRE prune/remove before next round
ELSE violation
```

What our work can add beyond GUARDIAN:

1. Convert GUARDIAN's temporal graph evidence into a verifier-neutral event schema.
2. Add evidence-completeness checking: if source edge, timestep, or response provenance is missing, verifier returns `unknown` instead of `safe`.
3. Map graph-based anomaly verdicts to symbolic governance rules such as `remove before next round`, `do not use tainted response`, or `block corrupted communication path`.
